One smart layout, many decks

Each pair below is a real pitch-deck card (left) recreated (right) using a single Gamma cssFlex smart-layout variant plus card-level custom CSS only — no per-deck code. Same flexbox engine restyled into three very different aesthetics.

What the card CSS does: color (text, accents, backgrounds), typography (family, size, weight, spacing, case), and light decoration (borders / dividers, header bands, ✕ / ✓ ::before markers).
What it does not do: no display:grid, flex, or column rules. All structure — column counts, the 2×2, the two-column split — comes from the layout’s own cellSize option and Gamma’s native column node.

venture Sans · blue · stat rows

Hero — 3-stat row

Color + type only: blue eyebrow & stat numbers, navy title, uppercase labels, and 1px vertical dividers between cells. The 3-across row is the smart layout’s own cellSize — not CSS.

Problem — sentence stats

Same recipe as the hero, just sentence-case stat descriptions instead of short labels. Color + type + the cell dividers.

Traction — 2×2 grid

Color + type + “cross” dividers (1px borders on the right column and second row). The text/stats split is a real column node; the 2×2 is the layout’s cellSize. CSS draws dividers and recolors — it adds no grid.

Comparison — banded table

The most decorative card: 1px cell borders + radius, gray/blue header bands (backgrounds), row separators, and ✕ / ✓ markers via ::before. Still the layout’s own two cells — CSS only paints them.

thesis Editorial serif · cream · gold

Giant number

Font swap to serif, gold/ink colors, a cream card background, and center alignment (text-align + margin). One cell — no layout change.

Old way / new way

Serif font, gold eyebrows, ink titles, cream background, and one 1px divider between the two cells. The two columns are the layout’s cells.

Giant stat

Without us / with us

Color + type, plus a monospace font swap on the last two lines (the code). The two columns are the layout’s cells.
